Package Version Prefix

2023年1月11日 · 阅读需 3 分钟
Lucky Duck
front-end fans

今天在项目引入新的npm包，开发完功能打算提交的时候发现 package.json 文件有更改：

"@docusaurus/core": "2.2.0" ---> @docusaurus/core": "^2.2.0

可以看到文件更改仅仅是 包的版本号多了个 ^ 符号前缀

语义化版本控制规范​

这个符号很重要吗？ 答案是重要，它是 语义化版本控制规范 的一部分

我们首先要知道 版本号格式的组成 ： 主版本号.次版本号.修订号

版本号修改规则​

那版本号一般怎么递增的呢？

每个级别的版本号都有不同的 递增条件 ：

  • 主版本号：当你做了不兼容的 API 修改
  • 次版本号：当你做了向下兼容的功能性新增
  • 修订号：当你做了向下兼容的问题修正

版本范围语法​

当然有，有了 递增条件 ， 我们就可以根据 版本范围语法 来限制项目程序引入的依赖项或开发包的版本范围。

而版本号符号前缀就是版本范围语法

如果你认真观察过你的package.json文件的话，你一定会发现，有的依赖版本号前是^，有的是~，有的版本号前没有任何符号

    "@docusaurus/core": "2.2.0",
    "@docusaurus/preset-classic": "~2.2.0",
    "@mdx-js/react": "^1.6.22",
    "clsx": "^1.2.1",
    "prism-react-renderer": "^1.3.5",
    "react": "^17.0.2",
    "react-dom": "^17.0.2"

  • ^ 表示主版本号不变，次版本号和修订号会自动安装最新的。举个例子:

    "@mdx-js/react": "^1.6.22", 版本范围 >= 1.6.22 < 2.0.0

  • ~ 表示主版本号和次版本号不变，修订号会自动安装最新的。举个例子:

    "@docusaurus/preset-classic": "~2.2.0", 版本范围 >=2.2.0 < 2.3.0

  • 指定版本，即无^、~。举个例子：

    "react": "17.0.2", 执行npm i安装依赖时，精准安装17.0.2版本，不会自动安装最新版本

更多参考 版本范围语法示例

版本控制的意义​

采用官方的一句话：如果依赖关系过高，可能面临版本控制被锁死的风险（必须对每一个依赖包改版才能完成某次升级）。而如果依赖关系过于松散，又将无法避免版本的混乱（假设兼容于未来的多个版本已超出了合理数量）。

版本控制作为这个问题的解决方案之一

Git ChangeId

Lucky Duck
front-end fans

今天提交代码git commit、git push之后发现代码无法推送至远端Gerrit服务器。以下是报错信息：

Error: missing Change-Id in commmit message footer

怎么会没有Change-Id，这玩意不是会自动生成的吗？看到接下来的提示：

Hint: To automatically insert Change-Id, install the hook:
gitdir=$(git rev-parse --git-dir); scp -p -P 29422 xxx@xx.xx.com:hooks/commit-msg ${gitdir}/hooks/

执行上面的命令后按照提示 git commit --amend ，不编辑直接退出文件。 然后执行 git log 发现Change-Id已经在本次提交自动生成了。

这是什么命令，为什么有效？

网上查到命令的解释是：

执行上面的命令，你的本地仓库的 .git/hooks 文件夹下的 commit-msg 文件将会被 Gerrit 的 commit-msg 文件覆盖掉，而它的作用是今后的提交当中也会自动生成一个 change-id。

参考链接:Gerrit的基本使用-如何给本地仓库添加钩子 commit-msg

Lucky Duck

Docusaurus 博客功能 由 博客插件 提供支持。

只需将Markdown文件（或文件夹）添加到 blog 目录。

博客作者列表可以添加到 authors.yml 。

博客文章标题下方的日期是从文件名中提取的，例如：

  • 2019-05-30-welcome.md
  • 2019-05-30-welcome/index.md

则该页博客会被认为是在 2019-05-30 这个时间写的。

博客文章文件夹可以方便地共同定位博客文章图像：

博客也支持标签！

如果你不想要博客一栏: 删除 blog 目录, 并在你的Docusaurus配置文件使用 blog: false。
